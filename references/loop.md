# 闭环流程与各模块职责

## 驱动方式（通用智能体）

**闭环由运行环境中的智能体/LLM 驱动**：智能体按「进化环」读取输入、执行动作、写入输出，即可形成无限进化。不依赖特定平台（Cursor/Claude 等）。  
**完整工作流**（每阶段读什么、写什么、执行清单）见 **[references/agent_evolution_workflow.md](agent_evolution_workflow.md)**。

## 闭环总览（与工作流对应）

```
假设 → 自主决策 → 自主执行 → 自主校验审核 → 自主优化反思 → (回到假设)
```

与原有五模块对应：假设＝主动假设；自主决策＝主动规划/决策；自主执行＝任务追踪+执行；自主校验审核＝完成校验；自主优化反思＝主动决策+吃一堑长一智。

## 各模块输入输出（概要）

| 模块           | 输入                     | 输出 |
|----------------|--------------------------|------|
| 假设           | 历史失败、能力缺口、私域 | 需求假设、能力缺口清单 |
| 自主决策       | 假设、能力清单           | 实施路线、next_action |
| 自主执行       | 实施路线、任务列表       | 脚本/文档变更、日志 |
| 自主校验审核   | 任务状态、完成物         | 通过/不通过、self_verify_result |
| 自主优化反思   | 校验结果、失败记录       | 下一轮假设/规划、教训、state 推进 |

## 防迷失检查点

- 每轮开始：读 `state/current_mission.json`，确认「当前使命」与「当前阶段」。
- 每轮结束：写回状态与日志，便于下一轮或其它智能体接续。

## 多轮连续执行

智能体可在一轮内执行多步（假设→决策→执行→校验→反思），也可**连续多轮**不暂停：完成一轮反思后直接进入下一轮假设，无需等待用户输入，实现自主进化。

## 可选：无人参与时的轮次推进

当没有智能体在跑工作流时，可用 **闭环跑者** 仅做轮次与日志推进（不替代智能体的假设/决策内容）：  
`python scripts/loop_runner.py` 一轮；`python scripts/loop_runner.py --daemon --interval 300` 常驻。  
真正的进化内容仍由智能体按 agent_evolution_workflow.md 执行。
