# 闭环流程与各模块职责

## 闭环总览

```
主动假设 → 主动规划 → 任务追踪 → 完成校验 → 主动决策 → (回到假设或规划)
```

## 各模块输入输出

| 模块       | 输入                     | 输出 |
|------------|--------------------------|------|
| 主动假设   | 用户补充、历史失败、私域 | 需求假设清单、能力缺口 |
| 主动规划   | 假设清单、能力缺口       | 能力清单、实施路线、任务分解 |
| 任务追踪   | 实施路线、任务列表       | 任务状态、当前进行项、阻塞项 |
| 完成校验   | 任务状态、完成物         | 通过/不通过、缺口列表、测试结果 |
| 主动决策   | 校验结果、失败记录       | 下一轮假设/规划/执行、教训记录 |

## 何时进入下一阶段

- **假设→规划**：假设稳定或用户已补充。
- **规划→追踪**：路线已分解为可执行任务。
- **追踪→校验**：有任务标记为完成或阶段结束。
- **校验→决策**：校验完成。
- **决策→假设/规划**：未满足则更新假设或重新规划；满足则结束或进入下一轮需求。

## 防迷失检查点

- 每轮开始：读 `state/current_mission.json`，确认「当前使命」与「当前阶段」。
- 每轮结束：写回状态与日志，便于下一轮接续。
