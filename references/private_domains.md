## 办公平台 · ihaier

- **类型**：桌面端应用（非浏览器内网页）。
- **识别特征**：
  - **图标**：蓝白配色，蓝色背景、中间为白色大字母「H」，圆角方形的应用图标（任务栏/桌面可据此辨认）。
  - **定位**：办公平台，内置 IM（即时消息），左侧导航含「消息」「联系人」「工作台」「视频会议」「云文档」「日历」「任务」「审批」等，消息入口常有红点/未读数字角标。
  - **企业应用**：海尔内部各类企业应用（内部系统、审批、IT 服务台等）大多可在 ihaier 内通过搜索或工作台入口找到并打开。
- **主界面布局（三栏）**：
  - **左侧导航**：工作台、视频会议、云文档、日历、任务、邮箱、机器人、收藏、**消息**、会议室预订、审批、II服务台、更多等；顶部有搜索「Q 职业 (Ctrl+K)」和「+」。左侧「**消息**」旁若有**红色圆圈+数字**，表示全部会话的未读总数（如红 8 = 共 8 条未读）。
  - **中间消息列表**：为会话列表，每条显示头像、会话名、消息摘要、时间。**列表最上面是一些置顶消息**，第一条可能是**当前用户自己**（如「我的」或本人会话）。非置顶部分按时间排序，最新在上。**带红色数字的条目 = 该会话有未读消息**（红 1、红 9 等即该会话未读数）。
  - **右侧详情**：点击中间列表某一项后，右侧显示该会话的**完整内容**——顶部为对方名称（如「周小帅」）及「文件」「文档」标签；中间为消息正文、链接、附件（如 .zip）；底部为输入框和发送/附件按钮。**右侧消息对话框中，最下面的一条消息是最新的**；消息时间显示在**对话框中间**（每条消息的时间在气泡或消息行中间位置）。要查看「谁找我、什么事」时，点开中间列表里带红字的会话即可在右侧看到详细信息。**右侧消息对话框里的某条消息可右键点击**，会弹出菜单（如**复制**等）。**发送消息**：在输入框处用 **Ctrl+V 粘贴**内容（执行前由调用方设置剪贴板，或 do 输入中文）；**发送方式**为点击**右下角发送按钮**或按 **Enter 回车键**。
- **未读与「有人找我」**：左侧「消息」红数字 = 有人找我（有未读）；中间列表某条旁的红色数量 = 该会话未读。查「是否有人找我」时，截图后 vision 可识别：左侧消息红数字、中间列表中各条红数字；若需补充到私域或汇报详情，可对带红字的会话执行「点击 → 截图右侧详情 → vision 提取摘要」。
- **能力与场景**：ihaier 的**能力**（如读取消息列表、消息详情、视频会议、云文档、会议室预约、工作台、审批、日历、任务、搜索、发消息等）与**场景**（如「谁找我」「发消息给某人」）的区分及执行链见 **references/ihaier_capabilities.md**。
- **窗口标题**：ihaier 主窗口在任务管理器中显示为 **「办公平台」**，激活时用 `window_tool activate "办公平台"`；进程名为 iHaier2.0（32位），也可用 `window_tool activate_process iHaier2.0`（部分环境按标题激活更稳）。
- **示例需求**：用 ihaier 给周小帅发消息。执行链：打开/聚焦 ihaier（`window_tool activate "办公平台"` 或 `activate_process iHaier2.0`）→ 截图 → 多模态 vision 识别「联系人/搜索/输入框」位置 → run_plan 执行 click、type 完成输入与发送；「谁找我」场景 = 读取消息列表（+ 可选消息详情）能力 + vision 判未读/摘要。
- 可补充：入口、登录方式、常用功能与菜单、术语、审批流、报表等。
- 闭环中的「主动假设」可纳入与 ihaier 相关的用户需求。
