# 项目约束与要求（必须遵守）

**设计宗旨**（作用、目标、方法、核心、注意、日志、UI、防迷失）见 **SKILL.md 中「本技能宗旨与约束」**，通用智能体每次驱动本技能时应先读该节再执行闭环。

以下为项目级要求，自我进化过程中不得违背。

1. **独立自闭环**  
   本技能不依赖 hex-machine-code-x64-win 等外部技能。截图、鼠标、键盘、屏幕、多模态视觉等能力均在本技能内自包含（scripts/ + 配置）。

2. **UI 形态**  
   CS（桌面客户端）与 BS（浏览器端）的优劣与难易已分析并做出决策，见 [references/ui_iq.md](ui_iq.md)。后续按该决策实现与迭代。

3. **私域知识命名**  
   私域知识不限于单一平台，故不使用 ihaier.md 作为文件名。统一使用 `references/private_domains.md` 存放多类私域（如办公平台 ihaier、其他业务域等），按需加载。

4. **骨架完成后即启动环路**  
   整体骨架（技能结构、脚本、references、UI、需求记录）完成后，立即启动自我进化闭环。闭环由**通用智能体**按 [references/agent_evolution_workflow.md](agent_evolution_workflow.md) 驱动：假设 → 自主决策 → 自主执行 → 自主校验审核 → 自主优化反思 → 回到假设，并持续运行。

5. **开放性项目**  
   在符合上述约束的前提下，不逐项向用户确认，按当前世界知识与常识自行决策、推进与迭代。

6. **要求记录**  
   上述要求必须记录在案（本文档），便于长期演进与多轮对话中不遗忘。

7. **多轮持续、不结束**  
   自闭环只做一轮不足以达成目标。必须**自主假设、自主规划、自主决策**多轮持续：每轮 decide 后更新 state 并进入下一轮假设/规划，**不得在一轮后向用户做「总结并结束」**。用户建好 UI 后应能持续看到自闭环整体进度（当前使命、阶段、轮次、近期日志）。
